# Туроператор: турпакеты

Учебный веб-проект по теме **«Туроператор: турпакеты»**.  
Сайт реализует базовый функционал информационной системы туроператора:

- хранение турпакетов, направлений, дат выезда;
- регистрация и авторизация пользователей;
- бронирование туров клиентами;
- обработка заявок менеджером;
- управление пользователями и правами доступа администратором.

Проект выполнен на **PHP + MySQL (MariaDB) + HTML/CSS/JS** и предназначен для локального запуска (например, в XAMPP).

---

## Функционал системы

### Роли пользователей

- **Клиент**
  - регистрация и авторизация;
  - просмотр списка турпакетов и подробной информации о туре;
  - выбор даты выезда и оформление заявки на тур;
  - просмотр списка своих заявок и их статуса.

- **Менеджер туроператора**
  - просмотр всех заявок клиентов;
  - изменение статуса заявки: `new`, `confirmed`, `paid`, `cancelled`;
  - привязка обработчика (менеджера) к заявке.

- **Администратор системы**
  - просмотр списка пользователей;
  - смена роли пользователя (client / manager / admin);
  - базовый контроль над корректностью данных и правами доступа.

---

## Используемые технологии

- **Backend:** PHP 8.x (подходит и 7.4+ для XAMPP)
- **База данных:** MySQL / MariaDB
- **Интерфейс:** HTML5, CSS3
- **Клиентский JS:** нативный JavaScript (без сторонних библиотек)

## Структура базы данных (кратко)

Основные таблицы:

* `roles` — роли пользователей (`admin`, `manager`, `client`);
* `permissions`, `role_permissions` — права и привязка прав к ролям;
* `users` — пользователи системы (логин, пароль, ФИО, телефон, email, роль);
* `auth_log` — лог входов в систему (успешных и неуспешных);
* `destination` — направления тура (страна, город, описание);
* `tour_package` — турпакеты (название, описание, маршрут, базовая цена, длительность);
* `tour_date` — даты выезда по турпакетам (доступные места, цена, статус);
* `payment_method` — справочник способов оплаты;
* `orders` — заявки клиентов на туры (кол-во участников, предпочтения, статус, сумма);
* `payments` — информация о платежах (упрощённо);
* `review` — отзывы клиентов по завершенным поездкам;
* `admin_setting`, `backup_log`, `email_queue`, `integration` — служебные таблицы.

В таблицах `users` и `orders` настроены триггеры на уровне БД, которые проверяют:

* формат логина (латиница + цифры, минимум 6 символов);
* формат телефона (строго `8(XXX)XXX-XX-XX`);
* корректность email.

## Установка и запуск (локально, XAMPP)

### 1. Установка окружения

1. Установить [XAMPP](https://www.apachefriends.org/) или иной стек (Apache + PHP + MySQL).
2. Запустить **Apache** и **MySQL** через XAMPP Control Panel.

### 2. Создание базы данных

1. Открыть в браузере: `http://localhost/phpmyadmin/`.

2. Создать новую базу данных, например:

   * Имя: `tour_operator`
   * Кодировка: `utf8mb4_general_ci`

3. Выбрать созданную базу слева в phpMyAdmin.

### 3. Импорт структуры БД

1. Перейти во вкладку **Import**.
2. Выбрать файл `tour_operator.sql` из папки `sql/` проекта.
3. Нажать **Go** / «Вперёд».
4. Убедиться, что таблицы успешно созданы (появится список таблиц: `users`, `tour_package`, `orders`, и т.д.).

### 4. Размещение файлов проекта

1. Скопировать папку проекта (например, `tour_operator_site/`) в каталог:

   ```
   C:\xampp\htdocs\
   ```

2. Убедиться, что путь выглядит примерно так:

   ```
   C:\xampp\htdocs\tour_operator_site\
   ```

### 5. Настройка соединения с БД

В файле `db.php` при необходимости указать свои настройки доступа к MySQL:

```php
$host = 'localhost';
$user = 'root';      // стандартно для XAMPP
$pass = '';          // пароль, если вы его задавали
$db   = 'tour_operator';
```

Если вы изменили имя БД или логин/пароль MySQL, поправьте значения в этом файле.

### 6. Запуск сайта

1. Открыть в браузере:

   ```text
   http://localhost/tour_operator_site/index.php
   ```

2. Должна отобразиться главная страница с каталогом туров.

## Создание учётных записей и ролей

### Клиент

1. Перейти на страницу регистрации: `http://localhost/tour_operator_site/register.php`.
2. Заполнить форму:

   * Логин: латиница и цифры, минимум 6 символов.
   * Пароль: минимум 8 символов.
   * Телефон: строго в формате `8(XXX)XXX-XX-XX`, например `8(900)123-45-67`.
   * Корректный email.
3. После успешной регистрации можно авторизоваться и создавать заявки.

### Первый администратор

Первый администратор создаётся вручную через phpMyAdmin:

1. Зарегистрировать обычного пользователя (он будет с ролью `client`).
2. Открыть phpMyAdmin → базу `tour_operator` → таблицу `users`.
3. Найти созданного пользователя.
4. Посмотреть таблицу `roles` и определить `id` роли `admin`.
5. В таблице `users` для этого пользователя изменить поле `role_id` на `id` роли `admin`.
6. После этого, при входе на сайт под этим логином, пользователь будет иметь права администратора и сможет менять роли другим пользователям через интерфейс **Админ-панели**.

### Менеджер

1. Администратор заходит в **Админ-панель**.
2. В списке пользователей выбирает нужного клиента.
3. Меняет роль на `manager`.
4. После этого менеджер получает доступ к **панели менеджера**, где видит все заявки клиентов и может менять их статусы.

   * просмотр списка пользователей;
